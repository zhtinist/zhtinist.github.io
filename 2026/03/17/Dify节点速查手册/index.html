1. 开始节点 Start

关键词：入口 开始 输入 参数 启动 start input 表单 用户输入

普通用法：添加输入变量

变量类型说明：

类型 用途
文本（String） 单行文字输入
段落（Paragraph） 多行长文本
数字（Number） 数值计算
下拉选项（Select） 枚举选项
文件（File） 上传 PDF、图片等

进阶用法

① 下游引用变量格式

例：{{#start.user_input#}}

② 设置默认值
在输入字段配置中填写「默认值」，当用户不填时自动使用该值。

③ 文件上传
类型选「文件」→ 可限制文件格式（PDF / 图片 / 文档）→ 下游用文档提取节点解析内容。

2. 结束节点 End

关键词：结束 输出 终止 end output 最终结果 workflow结束

普通用法：配置输出

进阶用法

① 输出多个变量
可同时添加多个输出变量，调用方通过 API 获取全部结果。

② 结构化输出
配合「结构化输出」功能，可将 LLM 输出限定为 JSON Schema 格式，便于程序解析。

⚠️ Workflow 类应用必须以结束节点收尾；Chatflow 类应用用「直接回复」节点，不强制要求结束节点。

3. 直接回复节点 Answer

关键词：回复 答复 输出文字 流式输出 answer reply streaming 实时输出

普通用法：输出对话回复

进阶用法

① 非终端输出（中间节点回复）
Answer 节点可放在工作流中间，在流程未结束时先输出部分内容（如”正在检索…”），提升用户体验。

② 流式输出控制
在 Answer 节点中引用 LLM 节点变量，可实现 token 级别的流式输出，字符逐字出现。

③ 迭代内嵌输出
在迭代节点内部放置 Answer 节点，每处理一条数据就实时输出一次结果。

4. LLM 节点

关键词：AI 大模型 GPT Claude 对话 生成 llm prompt 语言模型 问答 总结 翻译 写作

普通用法：调用 AI 模型

变量引用格式：

进阶用法

① 开启记忆（Memory）
适用于 Chatflow → 右侧面板 → 开启「记忆」→ 设置上下文窗口大小（对话轮数）→ LLM 自动携带历史对话。

② 结构化输出（JSON Schema）

③ 视觉能力（Vision）
选择支持视觉的模型（如 GPT-4o）→ 在用户输入中引用图片变量 → 模型可理解图片内容。

④ 角色切换
System Prompt 中设定角色人设，User 输入引用实际内容，实现角色扮演、客服、专家等场景。

⑤ 多轮对话上下文拼接
使用模板节点将历史消息手动拼接后传入 LLM，适合精细控制上下文长度的场景。

5. 知识库检索节点 Knowledge Retrieval

关键词：知识库 RAG 检索 搜索文档 向量 knowledge retrieval 文档问答 企业知识

普通用法：检索相关内容

下游引用：在 LLM 节点的 Prompt 中引用 {{#knowledge.text#}}。

进阶用法

① 检索模式选择

模式 适合场景
语义检索 语义相似，容忍措辞不同
全文检索 精确关键词匹配
混合检索 两者结合，效果最佳（推荐）

② 多知识库并行检索
添加多个知识库 → 系统自动合并结果 → 按相关性排序返回。

③ 重排序（Rerank）
开启 Rerank 模型 → 召回结果二次精排 → 提升准确性（需配置 Rerank API）。

④ 配合模板节点格式化检索结果

6. 问题分类节点 Question Classifier

关键词：分类 识别意图 路由 分流 classifier intent 意图识别 问题类型

普通用法：自动分类路由

💡 描述越具体越准确。例如：「用户询问产品价格、套餐费用、优惠活动等价格相关问题」比「价格」效果好 10 倍。

进阶用法

① 兜底分类
最后一个分类设为「其他 / 无法识别」→ 连接人工兜底或通用回复节点。

② 嵌套分类
第一层分类后，在某个分支内再加一个分类节点 → 实现二级意图路由（如：先分「售前/售后」，再分「技术/账单」）。

③ 配合变量聚合节点
多个分类分支处理完成后 → 接「变量聚合」节点 → 合并为统一变量 → 传入结束节点统一输出。

普通用法：添加多条件 OR / AND

不包含 字符串不包含
为空 变量为 null / 空字符串
不为空 变量有值
大于 / 小于 数值比较

③ 变量为空检测
判断某变量「为空」→ 走初始化分支（如首次设置语言偏好）→ 「不为空」走正常流程。

④ 数值区间判断（多 ELIF）

12. 变量聚合节点 Variable Aggregator

关键词：聚合 合并分支 汇总 aggregator merge 多分支合并 统一输出 收束

普通用法：合并多个分支为一个变量

进阶用法

① 问题分类后的聚合
问题分类节点将流量分为 3 个分支 → 各分支分别由不同 LLM 处理 → 3 个分支的输出接入同一个变量聚合节点 → 统一输出给结束节点。

② 条件分支后的聚合
IF/ELSE 各分支执行不同操作 → 最终聚合为同一变量 → 避免在结束节点重复引用多个变量。

15. 参数提取节点 Parameter Extractor

关键词：参数提取 结构化 抽取 提取信息 extractor 自然语言转结构 实体抽取 信息抽取

普通用法：从自然语言中提取结构化参数

示例：从「我想订明天去北京的机票」中提取：

参数名 类型 描述
date String 出发日期
destination String 目的地城市
type String 服务类型（机票/火车等）

进阶用法

① 提取后直接传入工具/HTTP 节点
提取出的结构化参数 → 直接映射为 HTTP 请求的 Query 参数或工具的输入。

② 配合 IF 节点验证提取结果
提取完成 → IF 节点判断关键参数「是否为空」→ 为空则提示用户补充信息。

18. Agent 节点

关键词：agent 自主 推理 工具调用 autonomous 多步推理 CoT 自主决策 智能体

普通用法：配置自主推理节点

进阶用法

① Agent + 工具组合
配置多个工具（网络搜索 + 代码执行 + 知识库）→ Agent 自主决定调用哪个工具。

② 作为 LLM 节点的扩展
将复杂多步任务拆解 → 前置步骤用固定节点处理 → 不确定步骤交给 Agent 节点。

③ 自定义 Agent 策略
开发 CoT / ToT / GoT 等自定义策略 → 上传为插件 → 在 Agent 节点中选用。

19. 工具节点 Tools

普通用法：添加内置工具

进阶用法

① 将 Workflow 发布为工具
工作流发布 → 「发布为工具」→ 其他 Agent / Workflow 可调用此工作流作为工具。

② 自定义工具接入
设置 → 工具 → 自定义工具 → 填写 OpenAPI Schema → 即可在工具节点中使用。

进阶用法

① 内容审核场景

② 审核人修改内容
审核人可在 Human Input 节点直接编辑 LLM 输出 → 修改后的内容传入下游节点。
